# Waviate Script VS Code Extension

This VS Code extension provides language support and development tools for Waviate Script shader files.

## Features

### 1. Language Support ✅
- **`.wc` files** - Associated with C language for Waviate sample processing
- **`.wcpp` files** - Associated with C++ language for advanced Waviate processing
- **`.wrs` files** - Associated with Rust language for Waviate shaders

### 2. Syntax Highlighting ✅
All three file types get proper syntax highlighting using standard C, C++, and Rust grammars.

### 3. Code Snippets ✅
Convenient snippets for common Waviate patterns:
- Function templates for `sample_process()` and `frequency_process()`
- Quick access to `WaviateSampleInput` fields
- MIDI and controller data access helpers

### 4. Autocomplete & IntelliSense ✅
Full autocomplete support for:
- Waviate API structs and types
- Function signatures
- Input structure fields (WaviateSampleInput, etc.)
- MIDI and controller interfaces

### 5. Hover Documentation ✅
Hover over Waviate API symbols to see inline documentation.

### 6. Auto-Generated C++ Properties ✅
When you open a `.wc` or `.wcpp` file:
- Automatically generates `.vscode/c_cpp_properties.json`
- Configures include paths pointing to your project's `Source/` folder
- Enables full IntelliSense for Waviate headers
- Forces the canonical Waviate.h to be included for symbol resolution

## Installation

### Step 1: Install Extension Dependencies
```bash
cd vscode-extension
npm install
npm run test
```

### Step 2: Headers go in Source/

The extension looks for `Waviate.h` and `Waviate.hpp` in your project's `Source/` folder. If they're missing, the extension will automatically offer to download them from GitHub.

That's it—the extension handles the rest!

### Package as VSIX
```bash
npx vsce package
```

## Usage

### Create a Waviate Shader

1. Create a new file with `.wc` extension:
   ```bash
   touch my_shader.wc
   ```

2. The extension automatically recognizes it as C and provides syntax highlighting

3. Type `waviate_sample` and press Ctrl+Space for the function template

4. Start coding your shader with full autocomplete!

### What Happens When You Open a Shader File

When you open a `.wc` or `.wcpp` file:
1. Extension checks for `Source/Waviate.h` — if missing, prompts you to download from GitHub
2. Generates `.vscode/c_cpp_properties.json` with proper include paths
3. IntelliSense automatically picks up `WaviateSampleInput` fields
4. Type `in->` to see autocomplete for all input structure fields
5. Hover over any symbol for instant documentation

## API Reference

### WaviateSampleInput
Core input structure for sample-wise processing:

```c
struct WaviateSampleInput {
    uint64_t samplesSinceAppStart;       // Absolute sample counter
    int32_t positionInBlock;              // Position in current block (0 to blockSize-1)
    int32_t blockSize;                    // Total samples in block
    int32_t inputChannelCount;            // Number of input channels
    int32_t outputChannelCount;           // Number of output channels
    float** inputDeviceSamples;           // Input audio [channel][sample]
    float** currentSampleData;            // Output buffer [channel][sample]
    float sampleRate;                     // Sample rate in Hz
    uint8_t* midiNoteOn;                 // MIDI note states [0-127]
    uint8_t* midiCCValue;                // MIDI CC values [0-127]
    float* controllerAxisValue;           // Controller analog values
    float** previousSamples;              // Historical output [channel][samplesAgo]
    // ... plus many more fields
};
```

### Function Signatures
```c
float sample_process(const WaviateSampleInput* in, WaviateSampleStateWriter* out);
float frequency_process(const WaviateFrequencyInput* in, WaviateFrequencyStateWriter* out);
```

If needed, you can manually download headers via the command palette:
- `Ctrl+Shift+P` → "Waviate: Download Headers"
- Downloads `Waviate.h` and `Waviate.hpp` from GitHub to your `Source/` folder
